# CITADEL_SETTINGS_AUDIT

## 1) Summary

- **Audit scope:** Read-only inspection of settings/configuration behavior under `apps/readest-app/src` plus task tracking docs.
- **Total settings/toggles/configurable behaviors found:** **101** (visible + hidden/internal, including boolean toggles, enums/selects, numeric controls, text inputs, and command-palette exposed settings/actions).
- **Most configurable areas:**
  - Reader settings dialog (`Font`, `Layout`, `Color`, `Behavior`, `Language`, `TTS`, `Custom`, `AI`)
  - Library/Home menus (`SettingsMenu`, `ViewMenu`)
  - Sync/account integrations (`KOReader`, `Readwise`, `Hardcover`)
- **Biggest UX issue observed:** the settings system is powerful but distributed across multiple surfaces (library dropdown, reader view menu, reader settings dialog, per-integration dialogs, command palette), which can feel scattered and overwhelming.

## 5) Citadel UX Opportunities

Potential simplification work using existing capabilities (no net feature loss):

- **Change defaults only**
  - Keep same-window opening as baseline (already aligned with current default).
  - Consider calmer defaults for advanced reader behavior (for new users) while preserving advanced controls.
- **Move toggles into calmer groups**
  - Promote a small “Reading essentials” subset; move less-used controls into deeper sections.
  - Consolidate window/device options currently split between library settings and reader controls.
- **Hide advanced controls behind “Advanced”**
  - Security (`Allow JavaScript`), custom CSS, low-level layout extremes (`maxInlineSize`, `maxBlockSize`) can be tucked behind explicit advanced affordances.
- **Reword ambiguous labels**
  - Clarify technical terms (“Single Section Scroll”, “Apply also in Scrolled Mode”, “Override Book Layout”) with plain-language helper text.
- **Add presets over many atomic toggles**
  - Reader presets: “Calm”, “Dense”, “E-ink”, “Research”.
  - Typography presets for common reading scenarios.
  - Sync behavior presets (off/manual/auto) layered on top of existing granular toggles.
